1. Project Overview

Motivation

Dengue outbreaks in southern Taiwan are driven by climate and mosquito dynamics, but early-warning tools remain limited. This project builds a forecasting system for Kaohsiung, Tainan, and Pingtung using multi-source data to support timely public health action. By integrating climate, rainfall, and mosquito indicators, the system provides early signals of rising dengue risk, enabling faster response and improved resource planning.

Objectives

  • Construct a unified weekly dengue–climate–mosquito dataset for southern Taiwan.
  • Compare four forecasting models: Random Forest, XGBoost, LSTM, LSTM–Transformer.
  • Identify key climatic and entomological drivers of dengue risk.
  • Deploy interactive dashboards using ArcGIS, Streamlit, and Supabase for decision support.

2. Methodology & Models

This project builds an end-to-end forecasting pipeline that links multi-source dengue, climate, and mosquito surveillance data to city-specific models for Kaohsiung, Tainan, and Pingtung. Models are trained on 2010–2021 data and evaluated on a held-out 2022–2024 test window.

2.1 End-to-End Forecasting Pipeline

The workflow has four main stages: (1) raw data ingestion from Taiwan CDC, mosquito inspections, NCHC climate feeds, and population statistics; (2) local ETL and preprocessing to build a clean weekly city-level time series; (3) model training and validation using Random Forest, XGBoost, LSTM, and LSTM–Transformer; and (4) cloud storage of model artifacts in Supabase, which are then visualized in a Streamlit web app deployed on Render.

2.2 Data Sources & Database Management

Four primary datasets are integrated: (1) weekly dengue cases from Taiwan CDC (2010–2024); (2) mosquito larval indices (BI, HI, CI) from field inspections; (3) daily meteorological and rainfall data from NCHC; and (4) city-level population density derived from official statistics and ArcGIS enrichment. All raw data are cleaned and merged locally in Python. After training, model outputs, such as predictions.csv, metrics.json, residuals.csv, history.csv, and feature_importance.csvare uploaded to Supabase Cloud Storage. This centralized, versioned store allows the Streamlit app to retrieve artifacts on demand without re-running models.

2.3 ETL, Cleaning & Feature Engineering

Daily dengue, climate, and mosquito data are aggregated to weekly city-level time series to match epidemiological reporting cycles. Missing case counts are treated as true zeros, meteorological gaps are filled via time-based interpolation, and mosquito indices are forward/backward filled between inspections. Implausible sentinel values are removed and temperature consistency constraints (Tmin ≤ Tmean ≤ Tmax) are enforced. For tree-based models, engineered lag features (1, 2, 4, 8, 10, 11, 12, 15 weeks) are created for cases, rainfall, temperature, humidity, windspeed, and BI/HI/CI. Continuous variables are standardized with z-scores for Random Forest/XGBoost and Min–Max scaled for deep learning models.

2.4 Modeling Setup

  • Task: One-week-ahead prediction of weekly dengue cases, with a separate model per city.
  • Data splits: Train 2010–2017, validation 2018–2021, test 2022–2024 (chronological, no shuffling).
  • Validation: Time-series cross-validation for Random Forest/XGBoost; fixed validation window with early stopping for LSTM-based models.
  • Metrics: MSE, RMSE, MAE, MAPE, and R² evaluated on the held-out test period.

2.5 Models & Architectures

Random Forest

Random Forest models were optimized using a time-series aware hyperparameter search. The search explored the number of trees (300–700), maximum tree depth (none, 15, 25), minimum split requirements, and feature subsampling ratios (0.7, 1.0, √p). Bootstrap sampling was enabled. The best configuration for each city was selected based on validation MAE.

XGBoost

XGBoost models were tuned using a targeted grid search designed for time-series forecasting. The search explored tree depth (2–8), learning rates (0.01–0.07), number of boosting rounds (500–800), and sampling ratios for rows (0.8–1.0) and features (0.8–1.0). A fixed random seed ensured reproducibility. XGBoost consistently achieved the lowest RMSE and highest R² across cities.

LSTM (Lag Features)

The LSTM (Lag Features) model first engineers 1–2 week lags of dengue cases, rainfall, temperature, humidity, windspeed, and mosquito indices (BI, HI, CI), then feeds standardized 24-week input windows into a stacked LSTM network. The model is trained on log-transformed cases using Adam optimization, early stopping, and sample weighting to give more importance to outbreak weeks.

LSTM–Transformer

The LSTM–Transformer model was tuned using a small, focused hyperparameter search designed for limited outbreak data. The search varied the input window length (4–16 weeks), number of attention heads (2–4), and transformer feed-forward dimension (64), while keeping the learning rate (0.001), dropout (0.0), and batch size (64) fixed. The combination with the lowest validation loss for each city was selected as the final architecture.

2.6 Deployment & Interactive Visualization

A Streamlit app, deployed on Render, serves as the main interface for exploring forecasts and diagnostics. The app connects to Supabase to load predictions, metrics, residuals, training history, and feature importance per city and model. Users can switch between Kaohsiung, Tainan, and Pingtung, compare Random Forest, XGBoost, LSTM, and LSTM–Transformer side by side, and inspect EDA, ACF/PACF plots, and residual behavior. This design separates heavy offline training from lightweight cloud delivery, making the system reproducible and easy to extend.

3. Key Results – Model Comparison

Across all three regions, tree-based models (Random Forest and XGBoost) consistently outperformed LSTM-based models in predicting weekly dengue cases, especially during large outbreaks. Below is a compact summary of how each model behaved by city and a cross-city performance table.

3.1 City-Level Performance

Kaohsiung City

Best: Random Forest RMSE 19.28 · R² 0.899

  • Random Forest captured the shape and magnitude of the 2023–2024 outbreak with the lowest overall error.
  • XGBoost performed well (RMSE 25.53, R² 0.822) but slightly underpredicted peak weeks.
  • LSTM and LSTM–Transformer produced overly smooth curves and underestimated the outbreak peak (RMSE > 47).

Tainan City

Best: XGBoost RMSE 113.88 · R² 0.928

  • Extreme 2015 and 2023–2024 outbreaks made performance differences most visible in Tainan.
  • XGBoost slightly outperformed Random Forest (RMSE 121.21, R² 0.918), closely tracking the steep epidemic curves.
  • LSTM and LSTM–Transformer failed to model the sharp peaks, with very large errors and negative R² values.

Pingtung County

Best: RF ≈ XGBoost RMSE ≈ 3.0 · R² ≈ 0.85

  • Lower, more irregular outbreaks still produced accurate forecasts from Random Forest (RMSE 2.95, R² 0.856) and XGBoost (RMSE 2.99, R² 0.851).
  • Tree-based models remained robust under low-count, noisy conditions.
  • LSTM and LSTM–Transformer underpredicted outbreak magnitude and showed unstable validation behavior.

3.3 Cross-City Insights

  • Tree-based models dominate: Random Forest and XGBoost achieved the best performance in every city, providing stable forecasts even with limited outbreak history.
  • Deep learning struggles with spikes: LSTM-based models systematically underpredicted sharp epidemic peaks, especially in Tainan, and often produced negative R² values.
  • Recent case lags drive predictions: cases_lag1 and nearby lags were consistently the most important features, with climatic and mosquito indices contributing more variably by region.
  • Operational takeaway: For southern Taiwan’s episodic, highly skewed dengue dynamics, tree-based ML models are currently more reliable than LSTM architectures for early-warning forecasts.

3.4 Key Visuals

Outbreak Pattern – Kaohsiung (2010–2024)

Weekly dengue cases in Kaohsiung show long periods of low transmission punctuated by large outbreaks, with major peaks in 2015 and 2023–2024. This motivates the need for early-warning models that can anticipate sudden surges.

Forecast vs Actual – Kaohsiung (XGBoost)

XGBoost closely tracks both the rise and decline of the 2023–2024 outbreak, capturing peak timing and magnitude more accurately than deep-learning models. This supports its selection as the primary operational model.

Model RMSE Comparison – Test Set

Across Kaohsiung, Tainan, and Pingtung, tree-based models (Random Forest and XGBoost) consistently achieve lower RMSE than LSTM-based architectures, especially in cities with large, abrupt outbreaks.

4. Interactive Tools

4.1 ArcGIS Online Dashboard

An ArcGIS Online dashboard was developed to visualize spatiotemporal dengue patterns, mosquito indices, and rainfall intensity across Taiwan from 2010–2025. This dashboard highlights the concentration of outbreaks in Southern Taiwan and supports exploratory analysis of space–time clustering.

A Streamlit web application lets users explore EDA, ACF/PACF, model performance, and forecast diagnostics for each city. Users can switch between Random Forest, XGBoost, LSTM, and LSTM–Transformer to compare test metrics and inspect predicted vs. actual time series.

5. SWOT Analysis

The table below summarizes the strengths, weaknesses, opportunities, and threats of the proposed dengue forecasting framework and its implementation in Southern Taiwan.

Strengths

  • Strong predictive performance from tree-based models.
  • Comprehensive 15-year dataset integrating dengue, climate, and vector indices.
  • Interpretable feature importance and diagnostic tools.
  • Interactive dashboards enabling exploratory analysis and model comparison.

Weaknesses

  • Missing and noisy mosquito and climate data requiring imputation.
  • Weekly resolution limits detection of rapid environmental changes.
  • City-level scale lacks finer neighborhood-level insights.
  • Few large historical outbreaks reduce model generalizability.
  • Short development timeline and free-tier cloud hosting restrict optimization and performance.

Opportunities

  • Integrate mobility, satellite, and land-use data for improved early warnings.
  • Adopt advanced spatiotemporal models (TFT, GNN, hybrid deep learning).
  • Increase spatial resolution for district- or village-level risk mapping.
  • Build automated pipelines and migrate to more scalable cloud infrastructure.

Threats

  • Climate change may alter outbreak patterns and reduce model stability.
  • Surveillance inconsistencies and reporting delays can degrade forecast accuracy.
  • Emerging serotypes or vector shifts may weaken model assumptions.
  • Human mobility can trigger unpredictable outbreak clusters.
